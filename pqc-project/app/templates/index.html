The Quantum Threat is Here

Welcome. This project demonstrates the need for Post-Quantum Cryptography (PQC) by running two parallel servers. While they share this user interface, their underlying security architecture is vastly different.

  • Server A (Classic Server): This server runs on standard TLS encryption (like RSA). It is strong by today's standards, but will be vulnerable to decryption by future quantum computers.
  • Server B (PQC-Enabled Server): This server uses PQC algorithms (Kyber and Dilithium) for both data-in-transit (TLS) and data-at-rest (database encryption). It is secure against both classical and quantum attacks.

Both servers are intentionally built with the same application-level vulnerabilities to demonstrate that PQC is a defense against cryptographic threats, not a replacement for secure coding practices.
